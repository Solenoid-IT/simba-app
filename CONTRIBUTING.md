## 🛠 How Can I Contribute?

### Reporting Bugs
* Check the [GitHub Issues](https://github.com/Solenoid-IT/simba/issues) to see if the bug has already been reported.
* If not, open a new issue. Clearly describe the problem and include steps to reproduce it.

### Suggesting Enhancements
We are always looking for ways to make Simba faster and more robust!
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful for the broader community, not just your specific use case.

### Pull Requests (PRs)
1. **Fork** the repository and create your branch from `main`.
2. **Code Style:** Ensure your code follows PSR-12 standards for PHP.
3. **Documentation:** If you add a feature or change an API, update the documentation (README or Wiki) accordingly.
4. **Consistency:** Keep the "Solenoid-IT" philosophy in mind: clean, scalable, and developer-friendly.
